## 1.3 Cryptographic Sovereignty

To ensure the Digital Centaur remains autonomous and immune to algorithmic enclosure by centralized arbiters of truth, the architecture utilizes:

- **Decentralized Key Management (Lit Protocol):** Threshold Multi-Party Computation (MPC TSS) combined with Trusted Execution Environments (TEEs). Instead of a single centralized private key, the network generates Programmable Key Pairs (PKPs) distributed across independent nodes, requiring a strict two-thirds consensus threshold to authorize decryption.

- **CRDT State Synchronization:** Conflict-Free Replicated Data Types (Yjs, Automerge) allow data to be modified concurrently across multiple offline or decentralized nodes without central coordination. When nodes reconnect, algorithms mathematically guarantee convergence to the exact same state.

- **Convergent Encryption:** Symmetric ChaCha20 applied within a peer-to-peer overlay network that strictly avoids Distributed Hash Tables (DHTs), organizing all document commits into a Directed Acyclic Graph (DAG) structure.

- **Access Control:** Gated content served through React `<BrowserOnly>` components, where the Lit JS SDK negotiates with the user's decentralized identity against programmable Access Control Conditions (ACCs), decrypting content directly within the local browser while preserving zero-trust architecture.

Sovereignty is cryptographically backed — the operator's digital state cannot be revoked, altered, or silenced by legacy institutions.
