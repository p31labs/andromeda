════════════════════════════════════════════════════════════════════════════════
                          🔐 SECURITY ARCHITECTURE
════════════════════════════════════════════════════════════════════════════════

Layer 1 - Transport
  • TLS 1.3 (mandatory)
  • HSTS headers
  • Certificate pinning optional

Layer 2 - Authentication
  • Cloudflare Access (JWT)
  • Legacy token fallback
  • WebSocket auth via query param

Layer 3 - Authorization
  • RBAC: admin, operator, reader, none
  • Per-conversation ACLs
  • Admin API protection

Layer 4 - Data Protection
  • Optional E2EE via MLS
  • AES-128-GCM message encryption
  • Ed25519 signatures
  • Forward secrecy (regular key rotation)

Layer 5 - Input Safety
  • JSON schema validation
  • SQL parameterization
  • XSS sanitization
  • Rate limiting (100 req/s per user)

                       🚀 DEPLOYMENT WORKFLOW

Prerequisites:
  □ Docker + Docker Compose
  □ Node.js 18+ + pnpm
  □ wrangler CLI installed
  □ Cloudflare account

Step 1 - Initialize Database:
  $ cd /home/p31/andromeda/04_SOFTWARE/k4-cage
  $ npx wrangler d1 create p31-telemetry
  $ # Copy database ID

Step 2 - Configure:
  $ # Edit wrangler.toml, set database_id = "xxx"
  $ npx wrangler secret put ADMIN_TOKEN
  $ npx wrangler secret put INTERNAL_FANOUT_TOKEN
  $ npx wrangler secret put STATUS_TOKEN

Step 3 - Deploy:
  $ ./deploy.sh    # Automated script
   OR
  $ npx wrangler deploy

Step 4 - Verify:
  $ curl https://k4-cage.trimtab-signal.workers.dev/health
  $ curl -X POST https://k4-cage.trimtab-signal.workers.dev/messages \
      -H "Content-Type: application/json" \
      -d '{"conversationId":"test","senderId":"will","content":"Hello"}'

The K⁴ mesh messaging infrastructure is now complete and ready for production
deployment at the Kingsland, GA facility. The system provides:

  • Real-time communication at edge latency
  • End-to-end encryption with MLS
  • Offline-first synchronization
  • Scalable to thousands of users
  • Privacy-preserving by design
  • Production monitoring and management

The architecture is built on Cloudflare's edge platform using Durable Objects
for actor-based isolation, D1 for persistent storage, and WebSocket/WebTransport
for real-time communication. The system can handle family and friends messaging
at scale while maintaining the strict privacy and security requirements of
P31 Labs' sovereign infrastructure.

All components are documented, tested, and ready for deployment.
