                              TECHNICAL ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

Storage Architecture:
  Primary:     D1 Database (SQLite on Cloudflare edge)
  Fallback:    KV Namespace (ephemeral key-value)
  Cache:       Durable Object storage (in-memory)

Communication Protocols:
  REST API:    HTTP/1.1 for CRUD operations
  WebSocket:   Real-time messaging and events
  WebTransport: Future high-frequency updates (ready)

Data Flow:
  Client → WebSocket/HTTP → FamilyMessagingDO
      ↓
    Validate & Store (D1)
      ↓
    Broadcast (WebSocket)
      ↓
  Recipients (Real-time)

Database Schema (6 tables):
  • messages          - Core message storage
  • conversations     - Conversation metadata
  • message_status    - Delivery/read receipts
  • typing_indicators - Real-time typing status
  • family_members    - Registered family members
  • family_relationships - Relationship graph

═══════════════════════════════════════════════════════════════════════════════
                              API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════

HTTP REST API:
  POST   /messages              → Send message
  GET    /messages/{convId}     → Get message history
  PUT    /messages/{id}/read    → Mark as read
  PUT    /messages/{id}/delivered → Mark as delivered
  POST   /conversations         → Create conversation
  GET    /conversations         → List conversations
  PUT    /conversations/{id}/typing → Update typing
  POST   /conversations/{id}/reactions → Add reaction
  GET    /messages/search       → Search messages

WebSocket Events:
  Client → Server:
    • message:send         - Send new message
    • typing:start/stop    - Typing indicators
    • message:read         - Mark as read
    • presence:update      - Update presence

  Server → Client:
    • message:new          - New message received
    • message:delivered    - Message delivered
    • message:read         - Message read
    • typing:indicator     - Typing status
    • presence:changed     - Presence update
    • message:reaction     - New reaction
    • connected            - Connection established

Deployment Steps:
  1. npx wrangler d1 create p31-telemetry
  2. npx wrangler d1 execute p31-telemetry --remote --file=schema.sql
  3. npx wrangler deploy
  4. npx wrangler secret put ADMIN_TOKEN
