# P31 Documentation Instructions

- Direct, clear language. No corporate jargon.
- Use the project's native vocabulary from `context.md` Section 6.
- When writing for external audiences, translate internal terms to plain language.
- Never use military, naval, or submarine metaphors.
- For grant writing: use HAAT framework (Human, Activity, Assistive Technology, Context).
- Reference FDA classifications when discussing regulatory status.
- Keep README files concise: what it is, how to run it, how to contribute.
