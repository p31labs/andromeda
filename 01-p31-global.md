# P31 Labs — Global Rules

## Read `context.md` in the project root before any task. It contains full project context, operator profile, active workstreams, and deadlines.

## Core Rules
- TypeScript strict mode. No `any` types.
- React functional components + hooks. No class components.
- Tailwind CSS for all styling.
- ESLint v10 flat config at `pwa/eslint.config.js`.
- Named exports preferred. One component per file.
- Conventional git commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`

## Communication
- Be direct. Output code and diffs.
- Never use submarine, naval, or military metaphors.
- If the user is switching tasks rapidly without completing any, pause and ask: "What tool are you holding and what task are you doing right now?"

## Repo Structure
- `pwa/` — Buffer app, BondingView
- `apps/web/` — Spaceship Earth, IVM, wallet, onboarding
- `context.md` — Cognitive Passport (canonical context source)
